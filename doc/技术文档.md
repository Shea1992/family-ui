## 3. 核心模块设计

### 3.1 数据模型

#### 3.1.1 TypeScript 类型定义

```typescript
// types/index.ts

// 性别枚举
export enum Gender {
  MALE = 'male',
  FEMALE = 'female'
}

// 关系类型枚举
export enum RelationType {
  PARENT_CHILD = 'parent-child',
  SPOUSE = 'spouse',
  SIBLING = 'sibling'
}

// 成员接口
export interface Member {
  id: string;
  name: string;
  nickname?: string;
  birthday?: string;      // ISO 格式: YYYY-MM-DD
  workplace?: string;
  photo?: string;         // base64 或 URL
  gender?: Gender;
  bio?: string;           // 简介
  createdAt: string;
  updatedAt: string;
}

// 关系接口
export interface Relation {
  id: string;
  sourceId: string;       // 源成员ID
  targetId: string;       // 目标成员ID
  type: RelationType;
  subType?: string;       // 具体关系：father-son, husband-wife 等
  createdAt: string;
}

// 家族数据容器
export interface FamilyData {
  members: Member[];
  relations: Relation[];
  version: string;        // 数据版本，便于迁移
  updatedAt: string;
}

// 图谱节点（用于 D3）
export interface GraphNode {
  id: string;
  name: string;
  photo?: string;
  gender?: Gender;
  x?: number;
  y?: number;
  vx?: number;
  vy?: number;
  fx?: number | null;
  fy?: number | null;
}

// 图谱连线（用于 D3）
export interface GraphLink {
  id: string;
  source: string | GraphNode;
  target: string | GraphNode;
  type: RelationType;
}
```

### 3.3 存储服务

#### 3.3.1 StorageService 设计

```typescript
// services/storage.ts

const STORAGE_KEY = 'family-data';

export class StorageService {
  // 加载数据
  static load(): FamilyData | null {
    const data = localStorage.getItem(STORAGE_KEY);
    return data ? JSON.parse(data) : null;
  }

  // 保存数据
  static save(data: FamilyData): void {
    localStorage.setItem(STORAGE_KEY, JSON.stringify(data));
  }

  // 导出为文件
  static exportToFile(): void {
    const data = this.load();
    if (data) {
      const blob = new Blob([JSON.stringify(data, null, 2)], {
        type: 'application/json'
      });
      const url = URL.createObjectURL(blob);
      const a = document.createElement('a');
      a.href = url;
      a.download = `family-data-${new Date().toISOString().split('T')[0]}.json`;
      a.click();
    }
  }

  // 从文件导入
  static importFromFile(file: File): Promise<FamilyData> {...}
}
```

## 4. 关键实现细节

### 4.1 力导向图优化

#### 4.1.1 性能优化
- **节点数量控制**：超过 200 个节点时启用简化渲染
- **节流处理**：拖拽和缩放操作使用节流函数
- **Canvas 渲染**：使用 2D Canvas 而非 SVG，提升大量节点性能

#### 4.1.2 交互优化
- **固定节点**：拖拽后固定节点位置（fx, fy），避免力模拟重新布局
- **聚焦动画**：搜索定位时平滑过渡到目标节点
- **层级展开**：点击节点只展开直接关联成员，避免一次性渲染过多节点

### 4.2 数据一致性

#### 4.2.1 删除成员时的级联操作
```typescript
const deleteMember = (id: string) => {
  // 1. 删除该成员参与的所有关系
  const relatedRelations = relations.filter(
    r => r.sourceId === id || r.targetId === id
  );
  relatedRelations.forEach(r => deleteRelation(r.id));

  // 2. 删除成员本身
  dispatch({ type: 'DELETE_MEMBER', payload: id });
};
```

```typescript
// 压缩图片并转为 base64
const compressImage = (file: File, maxWidth = 400): Promise<string> => {
  return new Promise((resolve) => {
    const reader = new FileReader();
    reader.onload = (e) => {
      const img = new Image();
      img.onload = () => {
        const canvas = document.createElement('canvas');
        const scale = maxWidth / img.width;
        canvas.width = maxWidth;
        canvas.height = img.height * scale;

        const ctx = canvas.getContext('2d')!;
        ctx.drawImage(img, 0, 0, canvas.width, canvas.height);

        // 压缩为 JPEG，质量 0.8
        resolve(canvas.toDataURL('image/jpeg', 0.8));
      };
      img.src = e.target?.result as string;
    };
    reader.readAsDataURL(file);
  });
};
```

## 8. 注意事项

### 8.1 性能注意事项
- 家族数据量大时，考虑分页加载或懒展开
- 图片统一压缩后再存储，避免 localStorage 溢出
- 力导向图在节点数超过 500 时性能下降，需考虑优化策略

### 8.2 数据安全
- 当前使用 localStorage 存储，数据仅限本地
- 敏感信息（如详细地址、联系方式）暂不存储
- 后续可扩展加密存储或后端同步

### 8.3 浏览器兼容性
- localStorage 有 5MB 限制，注意数据量控制
- Canvas API 在部分移动端浏览器可能需要前缀
- 文件导入导出使用 File API，需现代浏览器支持
