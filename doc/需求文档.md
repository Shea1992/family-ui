# 家族图谱 App 需求文档

## 1. 项目概述

### 1.1 项目名称
家族图谱（Family Genealogy Graph）

### 1.2 项目目标
构建一个基于知识图谱的家族关系可视化 Web 应用，以**力导向图**的形式展示家族成员间的**血缘关系和婚姻关系**，支持成员信息的录入、编辑和查询。

### 1.3 核心价值
- 将复杂的家族关系以直观的图形方式呈现
- 方便家族成员信息的统一管理和追溯
- 记录家族历史，传承家族文化

---

## 2. 功能需求

### 2.1 家族成员信息管理

#### 2.1.1 成员信息字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ID | string | 是 | 系统生成的唯一标识符 |
| 姓名 | string | 是 | 正式姓名 |
| 小名 | string | 否 | 日常称呼/昵称 |
| 生日 | date | 否 | 出生日期，格式：YYYY-MM-DD |
| 工作地 | string | 否 | 当前工作地或居住地 |
| 照片 | string | 否 | 支持图片上传，存储为 URL 或 base64 |

#### 2.1.2 成员信息操作
- **添加成员**：录入新成员的基本信息
- **编辑成员**：修改已有成员的信息
- **删除成员**：删除成员及其关联关系（需确认）
- **查看详情**：点击成员查看完整信息

### 2.2 家族关系管理

#### 2.2.1 关系类型

**1. 血缘关系**
- 父子关系（父亲 → 子女）
- 母女关系（母亲 → 子女）
- 兄弟姐妹关系（双向）

**2. 婚姻关系**
- 夫妻关系（双向，无方向）

#### 2.2.2 关系操作
- **添加关系**：在两个成员之间建立关系
- **删除关系**：移除已建立的关系
- **修改关系**：更改关系类型

### 2.3 力导向图可视化

#### 2.3.1 图形展示
- **节点**：每个家族成员对应一个节点
- **连线**：成员之间的关系用连线表示
- **力导向布局**：节点自动排列，关系紧密的成员距离更近

#### 2.3.2 节点样式
- 显示成员照片（圆形头像）或默认头像
- 节点下方显示姓名
- 可选：不同性别使用不同颜色边框区分

#### 2.3.3 连线样式
- 血缘关系：实线
- 婚姻关系：虚线或不同颜色
- 带箭头表示关系方向（配偶关系无箭头）

### 2.4 交互功能

| 功能 | 描述 |
|------|------|
| **拖拽节点** | 可自由拖动成员节点调整位置 |
| **缩放平移** | 鼠标滚轮缩放图谱，拖拽画布平移视图 |
| **节点展开/收起** | 点击节点展开或收起其关联成员 |
| **搜索定位** | 搜索成员姓名，自动定位并高亮显示 |
| **编辑功能** | 点击节点可编辑成员信息和关系 |
| **悬停提示** | 鼠标悬停显示成员基本信息 |

### 2.5 家族历史记录
- 支持为家族或成员添加历史事件、故事
- 时间轴形式展示家族发展历程（可选增强功能）

## 3. 非功能需求

### 3.1 部署方式
- **Web 在线版**：通过浏览器访问
- **本地数据**：JSON 文件存储数据

### 3.2 性能要求
- 支持至少 500 个成员节点流畅渲染
- 图谱交互响应时间 < 200ms
- 页面加载时间 < 3s

### 3.3 兼容性
- 支持 Chrome、Firefox、Edge 等主流浏览器
- 支持响应式布局，适配不同屏幕尺寸

## 5. 数据模型

### 5.1 成员（Member）

```typescript
interface Member {
  id: string;           // 唯一标识
  name: string;         // 姓名
  nickname?: string;    // 小名
  birthday?: string;    // 生日 (YYYY-MM-DD)
  workplace?: string;   // 工作地
  photo?: string;       // 照片 URL/base64
  gender?: 'male' | 'female';  // 性别（可选）
  createdAt: string;    // 创建时间
  updatedAt: string;    // 更新时间
}
```

### 5.2 关系（Relation）

```typescript
interface Relation {
  id: string;           // 唯一标识
  sourceId: string;     // 源成员ID
  targetId: string;     // 目标成员ID
  type: 'parent-child' | 'spouse' | 'sibling';  // 关系类型
  subType?: string;     // 子类型：father-son, mother-daughter, husband-wife 等
  createdAt: string;    // 创建时间
}
```

## 6. 用户故事

### 故事 1：添加家族成员
> 作为用户，我希望添加新的家族成员，输入姓名、小名、生日、工作地和照片，以便将家族成员录入系统。

### 故事 2：建立家族关系
> 作为用户，我希望在两个成员之间建立父母-子女关系或夫妻关系，以便正确展示家族结构。

### 故事 3：查看家族图谱
> 作为用户，我希望以力导向图的形式查看整个家族的关系网络，能够拖拽节点、缩放视图，以便直观地理解家族关系。

### 故事 4：搜索成员
> 作为用户，我希望通过姓名搜索家族成员，系统能自动定位到该成员在图谱中的位置，以便快速找到特定成员。

### 故事 5：编辑成员信息
> 作为用户，我希望点击图谱中的成员节点，能够编辑其信息和关系，以便维护数据的准确性。

## 8. 验收标准

1. 能够成功添加、编辑、删除成员信息

3. 力导向图正确展示所有成员和关系
4. 搜索功能能准确定位到目标成员
5. 所有交互功能正常工作

### 8.2 性能验收
1. 100个成员节点下图谱操作流畅
2. 页面首次加载时间不超过3秒
3. 数据保存和读取响应及时
